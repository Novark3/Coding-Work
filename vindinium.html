For Week 7 of the Computer Sciences Expeditions we choose to create AI's to compete against one another. These AI's were put into a competition using a game called Vindinium.

Vindinium is a game where people can make their own AI's and try to win competitions of getting mines, attacking one another, and getting the most gold. Everyone in our class took the time to code out their own bots in hopes that theirs would make it to the top so that they could brag to everyone.

The way that my bot is different from everyone else's is that my bot has a main board of what to do. Instead of having the robot scan through the entire code and having the conditions hiden amongst the code, I made a board full of if statements and else if statements. This makes it so that the robot only has to scroll through a small portion of code then execute the given function whether it be for health or attacking a certain player for mines.

if(bot.yourBot.life <= 30) {
health();
} else if((bot.findDistance(myPos, closestTavern) === 2) && (bot.yourBot.life < 80)) {
bot.goDir = bot.findPath(myPos, closestTavern);
} else if(freeMinesLeft > .4) {
mines();
} else if(percentMinesOwned > .65) {
health();
} else if(enemyMines.length > 0 && ownsMore == false) {
capEnemyMine();
} else if(ownsMore == true) {
strategicKill();

The way that this code works is that it scrolls through this block of code and checks if any of the statements apply to itself. For instance, the first line of code says that if the bot's health is equal to or less than 30 then activate the function "health." Which basiaclly means go to a tavern and drink until his health is full.

The 1st else if statement talks about how if the bot is next to a tavern and its health is lower than 80, than take a drink to top off the health.

These next else if statements are where things get interesting because the next three statements all involve capturing mines. The first else if statement in this group says that if there are more than 40% of mines left uncaptured then go capture them.

The second statement in this family states that if my bot owns 65% or more of all the mines on the field, then just stand next to a tavern and drink to keep your health at full.

The last statement in this family says that if the enemies have captured mines, go capture them for myself.

At the end of this block of code we have the code for killing other players for their mines since when you kill an enemy, you automatically take control of all the mines they owned.

At the first line of the "killing code," there is a statement that says, "If someone owns more mines than I do, go kill them and take all their things." Then the last two lines in the group just tell the bot to kill whoever is closest to him.
